DB_Result::tableInfo()

DB_Result::tableInfo() -- returns meta data about the result set

Description

mixed tableInfo (DB_Result $result [, mode $mode])

Parameter

  • $result - the result object to analyse

  • $mode - depends on database implementation

    • FrontBase, MS-SQL, MySQL, PostgreSQL

      • false (default) returns this array 
        [0]["table"]  table name
[0]["name"]   field name
[0]["type"]   field type
[0]["len"]    field length
[0]["flags"]  field flags

      • DB_TABLEINFO_ORDER returns this array 
        ["num_fields"] number of metadata records
[0]["table"]  table name
[0]["name"]   field name
[0]["type"]   field type
[0]["len"]    field length
[0]["flags"]  field flags
["order"][field name]  index of field named "field name"
        The last one is used, if you have a field name, but no index. Test: if (isset($result['meta']['myfield'])) { ...

      • DB_TABLEINFO_ORDERTABLE returns the same as above. But additionally 
        ["ordertable"][table name][field name] index of field named "field name"
        This is, because if you have fields from different tables with the same field name * they override each other with DB_TABLEINFO_ORDER.

    • InterBase, Informix, mSQL, Oracle8, ODBC, Sybase - No information

Returns

  • mixed - the result of the dropping query or DB_Error when fail
