Using the PHPDoc tool

Using the PHPDoc tool --  Calling the PHPDoc commandline program

Synopsis

phpdoc [ -h ] [ -? ] [ -f ] [ -h ] [ -i directory ] [ -s directory ] [ -d directory ]

Options

  • -h -?

    Display help/usage

  • -f

    Force the deletion of the destination directory for HTML/XML output file.

  • -i

    Directory with the PHPDoc include files. Normally the PHPDoc directory in your PEAR directory itself. See the 'Setup' section.

  • -s

    Directory where your source files reside.

  • -d

    Destination directory for the generated doc files.

Set up the program

The PHPDoc program phpdoc resides in <pearpath>/phpdoc/. Open this file and edit this section on the top of the file to match your system paths: 

/*** CONFIGURATION ***/

// Directory with PHPDoc include files
$phpdoc_include_dir = '/usr/lib/php/PHPDoc';
// directory where your source files reside
$source_files = '/usr/lib/php/';
// save the generated docs here:
$dest_apidoc_dir = '/usr/doc/pear';
// force the deletion of the target apidocs directory
$dest_apidoc_delete = false;

/*** END CONF ***/

You can set the parameters while calling this program, too. But it is recommended to edit at least $phpdoc_include_dir, because this usually never changes.

Unix specific settings

Check the first line to ensure it points to your php binary (installed as CGI, not as webserver module!) 
#!/usr/local/bin/php -Cq

Windows specific settings

For running the PHPDoc program on Windows, the PHPDoc directory contains a batch file phpdoc.bat. Edit the following lines to match the path of your PHP executable (not webserver module!). 
REM -------------------
set PHP_PATH=c:\php
REM -------------------
