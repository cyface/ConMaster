Contributing patches

The best possibility to contribute patches is to send them to the PEAR developers list.

Make sure your patch is based on a current CVS snapshot. Don't send the complete file(s). Use diff or the diff command of CVS and attach the output of diff to the mail.

Another possibility is to send the patches directly to the author(s) of the class. But sometimes the author isn't the maintainer anymore, on vacation or has no access to CVS at the moment. So better use the mailinglist.

Recommend way for generating patches: 

  • $ cd /cvs/php.net/pear/Net_Ping

  • Retrieve the latest version 
    $ cvs update

  • Make the modifications you want 
    $ vi Ping.php

  • Check if there have been no modifications while you was modifing the file. If there were, re-adapt your patch to this last release. 
    $ cvs update

  • Generate the patch against the latest version from CVS 
    $ cvs diff -u Ping.php > ping_diff.txt

Note for Outlook Express users:

The mailing list software accepts only attachments of the content type "text/plain". OE creates the content type of a attachment by checking the file ending. So make sure that the diff output, if stored in a file, the file has an ending, which is assigned to a program like Notepad (plain-text editors). So better rename your diff file to file-diff.txt before attaching it.
