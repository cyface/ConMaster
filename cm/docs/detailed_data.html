Overview of the Data Model

The ConMaster5 Data Model is based on previous incarnations of ConMaster that have proven themselves in production use. The model does not strive toward 3rd Normal Form, but instead strives for maximum ease of development and reporting, with performance an occasional consideration. Don't forget to picture the students and classes model when thinking about the relationships between teh main tables.

Quick Summary of all Tables

Contact A record of contact with a person.
Convention A particular instance of a convention (Genghis Con XII).
Event Something that can be participated in (game, seminar, etc.).
Group A group that people can belong to, such as the RPGA, or club.
Item Something that can be bought, perhaps at an auction, or awarded.
Location A room or table at a venue.
Note Messages attached to any other object.
Person An individual associated with the convention, whether they be staff, authors, attendees, judges, or vendors.
Person_Convention A registration for or assoication with a person with a con.
Person_Item A person purchasing an Item.
Person_Group A membership of a person in a group (and includes id number).
Person_Section An intersection (many to many) entity between Person and Section. It allows each section to be associated with many people, and each Person to be associated with many Sections.
Preference Holds a person's preferences (games, times, etc.)
Score_Packet A labeled subset of people participating in a particular event at a particular time. It is used to track the performance of that group of participants. Additionally, it is used to mimic the RPGA score packets that are used to sumbit performance data to the RPGA(tm) Network.
Section An occurance of an event at a specific time.
Sequence Numeric sequences for _id fields.
Time_Slot A block of time in which Sections for a particular con occur.
Transaction A Person paying (or being refunded) money.
Type A table to hold possible values for other fields.
Venue A hotel, game store, or other place where the con is held.

Score_Packet

Score_Packets represent a specific group of people participating in an event. Typically, there is one judge and multiple players, although the table can support other groupings (although there must always be one 'primary' judge, although this can be the organizer, or lead participant). The main purpose of this table is to enable processing of RPGA score packets, but it can be used as a generic grouping of participants. Often what we are calling a score packet is referred to as a "Table" at a con, for example, "I was at Bob's table." Since table has so many meanings, and is meaningless for many events, the notion of Score Packet was chosen, especially since many of the details tracked about the group of participants related to scoring.

Columns
Score_Packet_Id NUMBER pk
Parent_Section_Id NUMBER fk
Judge_Person_Id NUMBER fk
Number_Of_Players NUMBER (Count of non-judge participants used for error checking and pro-rating).
Total_Scenario_Score NUMBER  
Total_Group_Score NUMBER  
Total_Team_Score NUMBER  
RPGA_Event_Type STRING  
Date_Entered TIMESTAMP  
Date_Modified TIMESTAMP  
Scored_By_Person_Id NUMBER fk
Checked_By_Person_Id NUMBER fk
Entered_By_Person_Id NUMBER fk

Section

Sections belong to individual Events, and are thus unique to a single convention. Sections are, in Colorado parlance, the ".3" in "501.3". Sections tend to be most useful for RPGA role-playing events that run multiple times in at the same convention. If you have that same event running at 8AM, 1PM, and 7PM, each of those is a separate section. Note that a group of players in that event during that section is a Score_Packet, and that the time it runs at is a Time_Slot.

Type

Types are lists of values for the various places where such lists are useful, such as GUI choice lists. Right now the Name is stored in the destination table rather than the Id for ease of use, and since types rarely have much to them except their name, thus making links not useful.

Columns
Type_Id NUMBER pk
Group_Name STRING Name of the Type Group that this type belongs to. Note that there is no table of Type Groups - they are defined in this table, since they have no attribute other than name.
Name STRING The type itself - or a short version if applicable.
Long_Name STRING A longer version of the type code if applicable.
Value NUMBER If the type has an associated numeric value, it goes here.
Value_2 NUMBER A second possible associated numeric value.

Venue

Venue is a place where a convention is held, such as a hotel, or game store. Places where individual Events/Sections are held are Locations.
