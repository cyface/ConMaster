Introduction

ConMaster5 is an Open Source web-based system for managing conventions, especially gaming conventions. Conventions are organized gatherings of people at a common place to share knowledge, socialize, and have fun. Gaming conventions focus on playing games of various types to achieve these ends. ConMaster5 manages conventions by keeping track of the people associated with the convention, and what they are doing when. For small coventions, this is not a big deal, but for larger gatherings, managing these details becomes a full-time task that is made easier by effective software.

A brief note on the designation ConMaster5 - There have been 4 previous versions of convention management software release under the ConMaster moniker by Tim White and C/Face Design. These were for use in Denver, Colorado gaming conventions and for RPGA(tm) Network score packet management. These previous versions were built in ACI's 4th Dimension, and were not open source. ConMaster5 continues the ConMaster tradition, bringing it into a modern, and open source world.

At heart, ConMaster5 is very similar to a school registration system. It allows convention staff to track people, and the events they register for and participate in. These people might be simply participating, as students might, or they might be a judge, analogous to an instructor in our school analogy. For events that are scored, and a winner determined, these scores and placements are tracked, much like grades. Hopefully keeping this school model in mind will help you identify with the functions and data structures of ConMaster5.

ConMaster5 Goals

Many goals were set forth for ConMaster5, many as a result of previous ConMaster shortcomings.

  • Provide an easy-to-use convention management platform
  • Provide an open source option for distribution
  • Allow multiple developers to develop it with no special software
  • Require minimal client-machine installation
  • Be easily customizable by most end-users
  • Be easily extensible by skilled end-users
  • Be as platform-independent as possible, especially for the client
  • Encourage, but not require a proven convention management methodology
  • Be integratable into other gaming or convention related software
  • Be extremely easy to maintain, install and setup
  • Provide flexible archtecture for various usage scenarios
  • Allow multiple ConMaster5 users to share a single data repository
  • Provide flexible and easy-to-use import and export capabilities
  • Allow developers of varying skill and experience to contribute

ConMaster5 Architecture

To meet the ConMaster5 goals, many architectures were considered before the current architecture took shape.

The advantage of the current architecture is that it enables the majority of the ConMaster5 goals, but it is not without disadvantages, some of which are listed here:

    PHP is not often used in production corporate environments, and thus may not be a technology that contributors want to learn
  • PHP is going through a lot of revisions, and keeping up with the latest can be challenging
  • Free IDE's are immature, and commerical IDE's aren't cheap.

Despite these disadvantages, it remains, after much consideration, the best architecture to meet the stated goals.

High-Level Technology Overview

Conmaster5 uses a web browser at its client, in order to provide as much client-hardware & OS flexibility as possible, and to ease both self-service registration on the internet, and multi-user convention management access.

To enable a web browser as a client, you need a web server and we are using Apache. PHP actually will run under other web servers (IIS), but we are focusing on as portable a server-side solution as we can, and that means Apache. And to some degree, it's just insane not to use Apache.

So your web browser request hits Apache, and if the page name ends in .php, then it gets passed to the PHP extension. PHP parses the file, quickly compiles it, and then executes it. Although it is an interpreted language, PHP runs extremely fast, due to a great engine and smart caching algorithms. The parsed code will interact with the database, probably calling other .php files along the way. Once the database interaction is done, the results are merged with a .html template file and sent to the user. In this way the Business Logic and database interaction are separated from the HTML presentation. In fact, a user could use a GUI HTML editor such as FrontPage or DreamWeaver to build their own HTML forms for ConMaster, without writing a single lick of PHP code! (Or even, in most cases, looking at the HTML code view!). This plays nicely to our "let people with varying levels of skill contribute" goal.

Why a SQL database and not an object database? Mostly becuase object database performance, scalability, and reliability has yet to reach the level of SQL databases. Also, the SQL developer knowledge base is much larger, as is the availability of reporting tools.

ConMaster5 Tools

The choices of tools to support the goals are few - the open source requirement narrows the field considerably, as does the requirement that development tools be low (preferably no) cost. We've done our best to pick the best of the open source/free breed for these tools, which sometimes means the best support, not the most elegant tool.

  • Server
    • Apache Webserver - Pre-installed on most ISP's, free download
    • Hypertext Preprocessor - Pre-installed on most ISP's, free download
    • MySQL - Pre-installed on most ISP's, free download
  • Developer Client
    • DreamWeaver - Excellent GUI HTML editor, for creating HTML templates
    • FrontPage - Less Expensive GUI HTML editor, for creating HTML templates
    • PHPEdit - Open Source PHP IDE
    • CVS - Source Code control - only needed for those that want to commit changes to the master code
  • User Client
    • Web Browser - These days, probably IE 5+

ConMaster5 Roles

These are the various roles that people can take on while working with ConMaster5.

  • Developer - Codes at one or more layers of ConMaster5.
  • Installer - Installs a ConMaster5 system on one or more pieces of hardware.
  • Administrator - Sets up ConMaster5, and keeps it running on the servers.
  • Convention Coordinator - Sets up a convention in ConMaster5, controls permissions for it.
  • Event Coordinator - Adds/Updates/Deletes events.
  • Judge Coordinator - Schedules judges for events, looks at judge historical performance.
  • Pre-Registrar - Registers people for events and for the convention in bulk.
  • Registrar - Registers people for events and for the convention on-site, checks event availibility.
  • Scorer - Adds/Updates/Deletes Score Packets & updates People.
  • RPGA Liason - Imports RPGA event code data, exports Score Packet data to send to RPGA.
  • Publications Coordinator - Updates Event data and Extracts it for use in publications.
  • On-Line Registrant - Registers themselves for the convention and events on-line.
  • Availibility Checker - Checks the availibility/fullness of sections.
  • Reporter - Runs various reports on exisiting data.
  • Mailing List Manager - Checks Person table for duplicate entries, exports data for labels.
  • Award Ceremony Preparer - Generates Top Person/Judge reports, prints winner lists and certs.

ConMaster5 Sessions

These sessions are rough groupings of functionality that may correspond to high-level Session Beans, or groups of Session Beans.

  • Convention Set-Up
  • Add a new convention (possibly cloning a previous convention), Edit Descriptors for the convention.
  • Import Person Data.
  • Set up convention variables, such as payment types accepted, and slots.
  • Import event data (RPGA or other).
  • Assign Permissions to people (and link to add/update/deleting them).
  • Event Management
  • Find/Add/Update/Delete Events and Sections.
  • Import/Export Events in various formats.
  • Registration
  • Find/Add/Update/Delete People.
  • Register people for events (add Person Sections, update Sections).
  • Handle financial details of registration.
  • Print tickets, confirmation cards and badges.
  • Availibility Checker
  • Find and display clearly events, the number registered, and the capacity of Sections.
  • Score Packet Management
  • Find/Add/Update/Delete Score Packets and associated Person Sections.
  • Export Packets in RPGA Format and other formats.
  • Print Winner Lists, Top Player/Judge Reports, Certs.
  • Edit People's RPGA Numbers.
  • Judge Management
  • Assign judges to Sections.
  • Assign Sections to tables/locations.
  • View judge performance across conventions.
  • Enter comments about judges.
  • Record judge event preferences.
  • Reporting
  • Provide flexible reporting mechanism across tables.
  • Provided canned reports of various types.
  • Allow printing and exporting of reports.
  • Graphs?

ConMaster5 Screens

This is taken from previous versions of ConMaster, and may need to change.

  • Login Screen - User logs in with name and password, setting their permissions.
  • Splash Screen - Main screen that allows navigation to other main areas (sessions).
  • Session Search/List Screen - Shows a listing of records appropriate to the session, allows the list to be searched, and sorted. Also allows a record in the list to be selected for editing, or deleting. Access to add record, import/export etc.
  • Session Edit Screen - Shows a single record (and possibly child records). Allows editing of appropriate data, including child records if appropriate.
  • Report Builder - allows construction of a report.
  • Report Selector - allows selection of a report to be run.
  • Advanced List Options Screen - Allows complex searches, sorts, custom exports.
  • Self-Registration Screen - allows you to register yourself for the convention and events.

ConMaster5 Shared Modules

These are code modules that are used by more than one session.

  • Print - prints to a local printer using graphical formats.
  • List Manager - manages lists of Entities, allowing search, sort and drill to detail.
  • Import/Export - imports and exports Entities as text files to the client system.
  • Convention Information/Code Lookup - Lookup information from the Con Info and other codes tables.
  • Security Manager - Makes sure users see only what they are authorized to see.
  • Module Manager - Allows ConMaster modules to be activated/deactivated.
